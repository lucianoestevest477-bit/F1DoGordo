# UI Cockpit Design

## Direcao visual

A interface deve parecer um volante de F1 interativo usado para controlar um rack de audio.

Elementos principais:

- Volante/cockpit F1.
- Preto denso e premium.
- Fibra de carbono fake desenhada via JUCE Graphics.
- Metal escuro, gradientes e highlights.
- Botoes neon premium.
- Grips ovais e laterais fisicas.
- Vazios internos nos grips.
- Hub central dominante.
- Meters embutidos no hardware.
- Top bridge como peca fisica, nao topbar comum.

## Nao copiar referencias

As imagens de referencia servem apenas para estrutura e sensacao. Nao copiar:

- Logos.
- Marcas.
- Textos.
- Simbolos.
- Emblemas.
- Layout exato.
- Posicao exata dos botoes.

## Controles e legibilidade

Mesmo com visual agressivo:

- Labels precisam continuar curtos e claros.
- Valores precisam ser legiveis.
- Botoes precisam continuar clicaveis.
- Controles principais nao devem ficar escondidos.
- Tamanho minimo 1200x700 deve continuar usavel.

## Hardware mapping

A UI deve antecipar um uso com volante/controlador fisico:

- Um botao fisico = uma acao clara.
- Macros globais sempre visiveis na pagina GLOBAL.
- Botoes de modulo com nomes simples.
- IDs APVTS estaveis.
- Sem controles funcionais sem parametro real.
